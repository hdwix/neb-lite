# Documentation index

The `docs/` directory collects references for engineers building and operating Nebengjek.

- **Developer onboarding and conventions** (`developer-onboarding.md`): Start here for local setup, coding standards, testing expectations, and git workflow guidance.
- **Enum organization guide** (`enum-organization.md`): Rules for deciding where enums live and how to avoid cross-module coupling.
- **Ride creation-to-acceptance flow** (`ride-flow.md`): Sequence diagram for the ride lifecycle and guidance on driver selection behavior.
- **API collection** (`nebengjek.postman_collection.json`): Postman collection covering public Gateway endpoints.

Keep these documents up to date when workflows, conventions, or public contracts change.
